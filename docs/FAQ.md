## 安装与配置

### Q1: Windows端需要安装什么额外软件?

**A**: Windows端需要安装 **VB-Audio Virtual Cable**，用于采集系统音频。

下载链接: https://vb-audio.com/Cable/

安装步骤:
1. 下载 VBCable_Setup.exe
2. 右键 → 以管理员身份运行
3. 安装完成后重启电脑
4. 在FyAudio设置中选择 "VB-Audio Virtual Cable" 作为输入设备

---

### Q2: Android端为什么需要这么多权限?

**A**: FyAudio需要以下权限来实现完整功能：

| 权限 | 用途 |
|------|------|
| `RECORD_AUDIO` | 采集系统音频（需要MediaProjection） |
| `BLUETOOTH_CONNECT` | 连接蓝牙A2DP设备 |
| `INTERNET` | UDP网络通信 |
| `ACCESS_WIFI_STATE` | 检测WiFi连接状态 |
| `NEARBY_WIFI_DEVICES` | 扫描局域网设备（Android 13+） |

> 注意：音频采集需要额外的"屏幕录制"权限，这是Android系统的强制要求。

---

### Q3: 树莓派上运行需要什么型号?

**A**: 推荐以下型号：

| 型号 | 适用场景 | 备注 |
|------|----------|------|
| 树莓派 3B+ | 入门推荐 | 够用，性价比高 |
| 树莓派 4B | 性能推荐 | CPU更强，支持双屏 |
| 树莓派 Zero 2 W | 体积敏感 | 性能较弱，可做纯接收端 |

## WiFi连接

### Q4: 设备之间搜不到怎么办?

**A**: 按以下顺序排查：

1. **检查网络**: 所有设备必须在同一WiFi网络（同一网段）
2. **检查路由器**: 确认路由器开启了UDP广播/组播
   - 部分企业路由器默认关闭广播
   - 尝试在路由器设置中启用 "AP隔离关闭"
3. **防火墙**: 关闭Windows Defender防火墙或添加规则放行UDP 5001-5003端口
4. **重启**: 重启FyAudio应用

**Windows防火墙规则**（管理员PowerShell）:
```powershell
New-NetFirewallRule -DisplayName "FyAudio UDP" -Direction Inbound -Protocol UDP -LocalPort 5001,5002,5003 -Action Allow
```

---

### Q5: 为什么延迟有时候很高?

**A**: 可能原因及解决方案：

| 原因 | 解决方案 |
|------|----------|
| WiFi信号弱 | 将设备靠近路由器 |
| 网络拥塞 | 减少其他设备带宽占用 |
| 路由器性能 | 升级路由器或使用有线连接 |
| 蓝牙干扰 | WiFi 5GHz频段改用 |

**建议**:
- 优先使用5GHz WiFi
- 网关设备（树莓派）使用有线以太网
- 在FyAudio中查看RTT（往返延迟）数值

---

### Q6: 能否跨路由器使用?

**A**: 暂不支持。FyAudio依赖UDP广播，只能在同一局域网内工作。

如需跨网络：
- 使用VPN创建虚拟局域网（如 Tailcale、ZeroTier）
- 或通过有线连接将路由器桥接

## 音频同步

### Q7: 多设备同步精度是多少?

**A**: 在理想网络条件下：
- 同步精度: **30-80ms**（取决于网络延迟）
- 这意味着人耳基本无法感知不同步

> 注意：由于声速在空气中约340m/s，80ms的差异意味着声音到达人耳的距离差约27米，所以日常使用完全感知不到不同步。

---

### Q8: 同步延迟可以为零吗?

**A**: 技术上不可能为零，原因：
1. **网络传输**: 即使是同一局域网，也有几毫秒的传输延迟
2. **缓冲**: 需要缓冲来应对网络抖动
3. **编解码**: AAC编码/解码本身有延迟

实际最小延迟约 20-30ms（仅缓冲和编解码）。

---

### Q9: 手动延迟补偿怎么用?

**A**: 场景：不同蓝牙音响有不同的硬件延迟

调整方法：
1. 在"同步"Tab找到"手动延迟补偿"
2. 听到"咚"的一声时，用手指点击每台设备
3. 调整补偿值直到声音对齐

**经验值**：
| 设备类型 | 典型延迟 |
|----------|----------|
| 有线耳机 | 0-10ms |
| 普通蓝牙音响 | 50-150ms |
| 某些低端蓝牙音箱 | 200-300ms |

---

### Q10: 能否播放视频时同步?

**A**: 目前FyAudio仅同步音频，不支持视频。

如需视频同步：
- 这是业界难题（"唇音同步"问题）
- 需要专门的视频同步方案
- FyAudio未来版本可能支持

**当前建议**：
- 视频源设备直接播放（最低延迟）
- 其他设备作为纯音频接收

## 蓝牙相关

### Q11: 为什么Android端搜不到蓝牙设备?

**A**: 可能原因：

1. **权限不足**: 确保已授予 BLUETOOTH_CONNECT 权限
2. **设备未配对**: 先在系统蓝牙设置中配对
3. **设备不支持A2DP**: 确认蓝牙设备是音频设备
4. **Android版本**: Android 12+需要额外申请 NEARBY_DEVICES 权限

---

### Q12: 树莓派如何连接蓝牙音响?

**A**: 在树莓派上执行：

```bash
# 1. 进入蓝牙控制台
sudo bluetoothctl

# 2. 扫描设备
scan on

# 3. 配对（替换 XX:XX:XX:XX:XX:XX 为设备地址）
pair XX:XX:XX:XX:XX:XX

# 4. 信任设备
trust XX:XX:XX:XX:XX:XX

# 5. 连接
connect XX:XX:XX:XX:XX:XX

# 6. 退出
exit

# 7. 确保 bluealsa 运行
sudo systemctl status bluealsa

# 8. 如果没有运行，启动它
sudo systemctl start bluealsa
```

---

### Q13: 能否同时连接多个蓝牙音响?

**A**: **技术上可以，但 FyAudio 暂未实现**。

现状：
- Android/iOS 系统级限制，通常只能连接一个 A2DP 设备
- 树莓派通过 BlueALSA 可以实现多设备
- Windows/macOS 可以通过系统设置连接多个蓝牙音频设备

**替代方案**：使用3.5mm音频分线器或支持多输出的USB声卡。

## 故障排查

### Q14: 应用崩溃或无响应

**A**: 按以下步骤排查：

1. **更新应用**: 确保是最新版本
2. **重启应用**: 完全关闭后重新打开
3. **重启设备**: 手机/电脑重启
4. **检查日志**: 
   - Android: `adb logcat | grep fyaudio`
   - Windows: 查看事件查看器
5. **清除缓存**: 设置 → 应用 → FyAudio → 清除缓存

---

### Q15: 音频卡顿或断续

**A**: 可能原因及解决：

| 症状 | 可能原因 | 解决方案 |
|------|----------|----------|
| 偶尔卡顿 | WiFi干扰 | 改用5GHz或有线 |
| 持续卡顿 | CPU占用高 | 关闭其他程序 |
| 长时间卡顿 | 缓冲不足 | 在设置中增加缓冲 |
| 单设备卡顿 | 该设备问题 | 重启该设备的应用 |

---

### Q16: 音源端停止播放但接收端还在响

**A**: 这是**正常行为**，说明同步在正常工作。

FyAudio 的工作原理：
1. 音源端采集音频并**缓存**到缓冲区
2. 接收端按时间戳**逐步播放**
3. 当缓冲区播放完毕后，接收端才会停止

如需立即停止：
- 在接收端手动点击"停止"
- 或断开网络连接

## 词汇表

| 术语 | 解释 |
|------|------|
| **Source** | 音源端，负责采集和广播音频 |
| **Receiver** | 接收端，接收并播放音频 |
| **Gateway** | 网关，树莓派专用，兼具接收和中继功能 |
| **A2DP** | 高级音频分发规范，蓝牙音频传输协议 |
| **UDP** | 用户数据报协议，无连接的网络协议 |
| **RTT** | 往返时间，网络延迟的度量 |
| **VB-Cable** | 虚拟声卡，用于Windows系统音频采集 |
| **BlueALSA** | 树莓派上的蓝牙A2DP服务 |
| **MediaProjection** | Android系统API，用于采集屏幕/音频 |
